# Glossary

Shared terms used across the handbook. Add entries as sections are written.

| Term | Definition |
|------|------------|
| **Control plane** | Components that manage cluster or system state (schedulers, controllers, APIs). |
| **Data plane** | Components that handle user or application traffic and data. |
| **Day-two operations** | Tasks after initial deployment: upgrades, scaling, incidents, backups. |
| **HA** | High availability — service continues during component failure. |
| **RPO** | Recovery point objective — maximum acceptable data loss window. |
| **RTO** | Recovery time objective — maximum acceptable downtime. |
| **SPOF** | Single point of failure. |
| **Sidecar** | Auxiliary container or process colocated with a main workload. |
| **Stateful workload** | Workload with persistent identity or storage requirements. |
| **GitOps** | Desired state declared in Git; controllers reconcile live infrastructure. |
