# Goals

## Purpose

This handbook is a practical, production-oriented reference for infrastructure tooling. Each section should help a reader choose, deploy, operate, and troubleshoot a tool in real environments.

## What we cover

- Tool overview and when to use it
- Architecture and core concepts
- Deployment patterns (bare metal, VM, container, Kubernetes)
- Configuration essentials
- Observability, backup, and security considerations
- Common failure modes and fixes
- Production readiness checklists

## What we do not cover

- Vendor marketing or feature lists without operational context
- Tutorial-only content with no production guidance
- Duplicating upstream docs; link to official references instead
- Application-level business logic

## Quality bar

- Accurate and version-aware where versions matter
- Actionable: commands, configs, and decision criteria
- Consistent structure across tool sections
- Honest trade-offs: when not to use a tool

## Audience

Platform engineers, SREs, and developers who own infrastructure decisions or day-two operations.
