# Dash0 synthetic monitoring — fundamentals

## Core idea

Synthetic monitoring proactively tests applications and endpoints on a schedule, generating traffic against critical paths rather than waiting for real users to hit a problem. By running scripted checks from outside the system, it catches outages, broken flows, and latency regressions before customers do, and establishes a baseline of availability and response time. It complements real-user monitoring: synthetics tell you the path is broken even when no one is currently using it.
