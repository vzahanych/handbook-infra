# Dash0 observability as code — Kubernetes-native and API

## Core idea

Configuration is applied through Kubernetes-native support and an API, giving two automation surfaces for the same definitions. The Kubernetes path lets dashboards and alerts be expressed as cluster resources and reconciled like any other manifest, while the API lets external tooling and pipelines push config programmatically. Together they make monitoring setup repeatable and automatable rather than a sequence of manual clicks.
